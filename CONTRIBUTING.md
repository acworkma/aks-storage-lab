# Contributing to AKS Storage Lab

Thank you for your interest in contributing to the AKS Storage Lab! This document provides guidelines for contributing to this project.

## Ways to Contribute

- **Report Issues**: Found a bug or have a suggestion? Open an issue!
- **Improve Documentation**: Fix typos, clarify instructions, or add examples
- **Add Features**: Enhance the lab with new scenarios or capabilities
- **Share Feedback**: Let us know how the lab can be improved

## Getting Started

1. Fork the repository
2. Create a new branch for your changes
3. Make your changes
4. Test your changes thoroughly
5. Submit a pull request

## Pull Request Guidelines

- Keep changes focused and atomic
- Update documentation to reflect your changes
- Test all scripts and templates before submitting
- Follow existing code style and conventions
- Provide a clear description of your changes

## Testing Your Changes

Before submitting a PR, please verify:

- All bash scripts execute without errors
- Bicep templates compile successfully
- Python code follows PEP 8 guidelines
- Kubernetes manifests are valid YAML
- Documentation is clear and accurate

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on what is best for the community
- Show empathy towards other community members

## Questions?

Feel free to open an issue for any questions about contributing!
